信息安全

五月 4, 2026 信息安全 本文总阅读量次

信息安全是一门交叉学科，设计多方面的知识。信息安全研究大致可以分为基础理论研究、应用技术研究、安全管理研究等。基础理论研究包括密码研究、安全理论研究；应用技术研究包括安全实现技术、安全平台技术研究；安全管理研究包括安全标准、安全策略、安全测评等。

网络安全术语

0 day: 漏洞未公布；

1 day: 漏洞公布，官方无补丁

n day：公布漏洞，官方有补丁

兔杀是免杀的口误或戏称。免杀是一种网络安全技术，指通过技术手段（如修改特征码、加壳、花指令等），让病毒、木马或恶意程序躲避杀毒软件的查杀。

POC

Proof of Concept，概念验证。

  • 定义：一段用来证明漏洞真实存在的代码 / 脚本。
  • 核心目的：验证漏洞是否可被利用，本身不做恶意操作，只是 “证明这里有个洞”。
  • 例子：写个脚本，能让目标服务器返回特定错误，证明 SQL 注入漏洞存在，但不会拿它来拿权限。

EXP

Exploit，漏洞利用。

  • 定义：利用系统漏洞进行攻击的动作 / 代码，是真正发起攻击的部分。

  • 和 POC 的区别：

    • POC = 证明漏洞存在（验证）
    • EXP = 利用漏洞达成目标（攻击）

  • 可以理解为：POC 是 “我知道这里有个洞”，EXP 是 “我要从这个洞钻进去”。

Payload

有效载荷。

  • 定义：EXP 成功执行后，真正在目标系统上执行的代码 / 指令，也就是你攻击的最终目的。
  • 例子：拿到服务器权限后，执行 “创建管理员账号”“上传木马”“窃取数据” 这些操作的代码，都属于 Payload。

Shellcode

  • 定义：Payload 的一种，专门用来在目标机器上获取Shell（命令行交互界面），因此得名 “Shell 代码”。
  • 作用：执行后会在目标机器上打开一个命令行窗口，让攻击者可以远程输入指令控制目标。
  • 分两种：
    • 正向 Shell：目标主动开放端口，攻击者连接过去。
    • 反向 Shell：目标主动连接攻击者的机器（更常用，因为不容易被防火墙拦截）。

工具

NetCat

NetCat (nc）是一个轻量、灵活的命令行工具，核心功能是通过 TCP/UDP 协议读写网络数据，功能全面。它既可以做网络调试工具（端口测试、数据传输），也能做渗透工具（反弹 Shell、搭建后门）。正向shell就是服务端把shell传给客户端使用，反向shell就是客户端把命令行窗口给服务端使用。（Kali内自带）

参数 作用 实际场景
-l 开启监听模式（作为服务端，等待连接） 接收反弹 Shell、搭建文件传输服务端
-p 指定本地端口 nc -lvp 4444：在本地 4444 端口开启监听
-t 以 telnet 形式应答 模拟 telnet 交互，适配老旧设备
-e 程序重定向（绑定 Shell / 程序） 正向 Shell 常用，nc -lvp 4444 -e /bin/bash
-n 纯 IP 形式通信，不做 DNS 解析 内网环境使用，避免解析失败
-v 显示执行过程 调试用，能看到连接建立、数据传输的细节
-z 扫描模式，不进行数据交互 端口扫描，比如nc -zv 192.168.1.1 1-1000
-u 使用 UDP 协议传输（默认 TCP） 测试 UDP 端口、DNS / 游戏服务调试
-w 设置超时时间（单位秒） 防止连接卡死，比如nc -w 3 192.168.1.1 80
-d 后台运行模式 隐藏进程，常用于搭建隐蔽后门

密码学

密码学主要研究数据加密算法、消息摘要算法、数字签名算法和密钥管理。加解密密钥相同的算法称为对称算法，典型的有DES、AES等；加解密密钥不同的算法称为非对称算法，通常一个密钥公开，另一个私藏，也称公钥算法，典型算法有RSA、ECC。

对明文的加密方式有两种：一种是逐位加密，成为流密码或序列密码；另一种是先对明文消息分组，再逐组加密，称为分组密码。

设计密码系统的两种基本方法：混淆和扩散。混淆指让密文与密钥的关系变得复杂，即使获取的明文和密文，也无法推出密钥；扩散指让明文中的每一位影响密文中的许多位，隐蔽明文的统计特征，挫败推测出密钥的尝试。

数据加密

数据加密核心是用算法+密钥把明文转成密文，防止数据泄露、篡改，算法主要分对称、非对称、哈希三类，常组合使用。例如，先用AES加密大数据，之后使用RSA/ECC加密AES密钥，然后传输，之后使用私钥解密密钥，最后用AES解密数据。

对称加密

对称加密是一种单密钥加密，特点是加解密同密钥、速度快、适合大数据，但密钥分发难。

1、AES

高级加密标准，目前最主流、最安全的对称加密算法，常用于文件、磁盘、HTTPS加密，首选AES-256。

分组长度：128位
密钥长度：128/192/256位
安全性：极高，目前暂无有效破解
典型用途：HTTPS传输加密、文件磁盘加密（Zip、7z、BitLocker）、数据库加密、VPN、WIFI、蓝牙

2、ChaCha20

流加密密码，不需要硬件加速也很快，多用于移动端、IoT，代表：WireGuard VPN、SSH、TLS1.3。

3、SM4

中国国密对称加密算法，分组128位，密钥128位，安全性对标AES。

4、DES/3DES

DES：分组长度为64b的密码算法，其中每8b有一位奇偶校验位，所以有效密钥长度为56位。该算法公开，安全性取决于密钥的保密程度，目前已被破解，不可靠。

3DES：三次DES，安全性一般，速度慢，目前逐步被淘汰。

非对称加密

非对称加密需要两种密钥，公钥和私钥，公钥加密，私钥解密。特点是安全性高，但速度慢。常用算法有RSA、ECC。常用于HTTPS/TLS、邮件加密、区块链签名、身份认证等场景。

1、RSA

目前最普及的非对称加密算法，基于大整数分解难题，常用长度2048/4096位。常用于HTTPS握手、数字签名、加密对称密钥（混合加密）、身份认证。缺点是密钥越长越慢，不适合加密大文件。

2、ECC

椭圆曲线加密算法，基于有限域上椭圆曲线离散对数问题（ECDLP）。密钥长度256位，安全性等同RSA-3072bit。特点是速度快、占用小，适合手机、IoT、区块链（比特币、以太坊）。常见算法有ECIES、ElGamal、X25519、ed25519。

应用场景包含：密钥交换（ECDH）、加密（ECIES）、签名（ECDSA、EdDSA 等）。
常见椭圆曲线：secp256r1、secp256k1、Curve25519、Edwards25519。

ECIES

椭圆曲线集成加密方案（Elliptic Curve Integrated Encryption Scheme），工业级标准 ECC 加密算法，是现在真正商用落地的 ECC 加密方案。

定位：

  • 非对称 ECC + 对称加密 + HMAC 鉴权
  • 既能加密任意字符串 / 二进制，又解决了 ElGamal 密文膨胀、不能直接加密消息的问题
  • TLS、国密、区块链、物联网都在用

核心特点：

  1. 不是直接用ECC加密明文。

    ECC运算慢、只能加密曲线上的点；

    ECIES用ECC协商会话密钥 → 对称 AES 加密明文，兼顾安全和速度。

  2. 自带 HMAC 完整性校验，防篡改。

  3. 密文膨胀极小，适合业务使用。

  4. 比 ECC-ElGamal 更实用、生产唯一首选。

ElGamal

基于离散对数难题的非对称加密算法，ECC-ElGamal 就是把它搬到椭圆曲线上实现。

（1）ECC-ElGamal 是什么？

  • 属于 ECC 加密体系
  • 功能：公钥加密、私钥解密
  • 安全性：和 256 位 ECC 等价 = 3072 位 RSA
  • 特点：加密结果不唯一（同一段明文，每次加密结果不一样）
  • 缺点：密文会膨胀 2 倍（工程用得少，理论教学多）

（2）核心原理

密钥生成：

  • 私钥：一个随机数
  • 公钥：私钥 × 椭圆曲线基点（ECC 固定点）

加密：

  • 用对方公钥生成一个临时密钥
  • 用临时密钥把明文点 “掩盖”
  • 输出两组数据：(临时公钥，密文点)

解密：

  • 用自己私钥还原临时密钥
  • 解开密文得到明文点

（3）ECC-ElGamal 优缺点

优点：

  • 安全强度高
  • 抗量子攻击潜力比 RSA 好
  • 同明文每次加密结果不同（安全）

缺点：

  • 密文膨胀 2 倍（不适合大文件）
  • 工程中不如 ECIES 常用
  • 不能直接加密字符串，必须映射成椭圆曲线点

EdDSA

Edwards25519，一种椭圆曲线数字签名算法Edwards-curve Digital Signature Algorithm (EdDSA)，基于扭曲爱德华曲线。

安全强度：约128位（等效于 RSA-3072）。
用途：生成、验证数字签名，用于SSH密钥、Git提交签名、TLS、加密货币等场景。
标准：RFC 8032；SSH 中标识为 ssh-ed25519

关键优势（对比 RSA/ECDSA）

  • 更快：签名 / 验签速度显著优于RSA、ECDSA。
  • 更安全：无随机数泄露风险，抗量子攻击能力更强。
  • 更轻量：密钥短（公钥 32 字节、私钥 64 字节），适合嵌入式 / 移动设备。

3、SM2

中国国密非对称算法，基于ECC改造，用于签名、密钥交换、加密。

哈希算法

利用哈希算法，可以将不定长的信息变为固定长度的摘要（指纹），信息的改变将引起摘要的改变，过程不可逆。因而算法常被用于检测信息是否被篡改、文件完整性校验、数字签名摘要、密码存储，典型算法有SHA-256、SM3、MD5(不可靠)。

单向散列函数又称哈希函数，是将任意长度的消息M映射成一个固定长度的散列值h。它的特点是：计算哈希值容易；反推明文消息难；找到具有相同哈希值的明文难。

1、SHA-2家族

SHA-256：256位摘要，最常用。
SHA-512：512位摘要，更高安全。

用途：密码存储（哈希加盐）、文件完整性校验、数字签名。

2、SHA-3家族

新一代安全哈希，抗碰撞性更强，目前不太普及。

3、Argon2

Argon2 是 2015 年 密码哈希竞赛 (PHC) 冠军，现代内存困难型 (Memory-Hard) 密钥推导 / 密码哈希函数 (KDF)，设计目标：抵抗 GPU/ASIC/FPGA 暴力破解、侧信道攻击、彩虹表。

（1）三个变种

  • Argon2d：数据依赖内存访问；最强抗 GPU/ASIC；易受侧信道攻击；适合离线 / 本地、加密货币。
  • Argon2i：数据独立内存访问；抗侧信道；抗 GPU 弱；适合在线服务、密码存储。
  • Argon2id（推荐）：前半 i、后半 d 混合；兼顾抗 GPU + 抗侧信道；RFC 9106 首选，通用场景默认。

（2）核心参数（决定安全与性能）

  • p（并行度）：线程数，默认 4
  • m（内存成本）：占用内存 (KB)，默认 65536（64MB）
  • t（时间成本）：迭代次数，默认 2
  • salt（盐）：随机 16 字节
  • hash_len：输出长度，32 字节（256bit）

（3）优点

  • 抗GPU极强：内存硬，GPU/FPGA 成本陡增
  • 抗侧信道：Argon2i/id 防护时序 / 功耗攻击
  • 可调节：按服务器性能调 p/m/t
  • 无专利：公共领域 / CC0，免费商用
  • 替代：MD5/SHA-256（不安全）、bcrypt（内存有限）、PBKDF2（易 GPU 加速）

（4）典型用途

  • 用户密码哈希存储（Web/App 首选）
  • 密钥推导（如从密码生成加密密钥）
  • API Key/Token 哈希
  • 加密货币钱包密钥保护
  • IoT / 低功耗设备密码存储

（5）推荐参数

  • p=4（线程）
  • m=65536（64MB）；金融级 131072–262144（128–256MB）
  • t=2；高安全 3
  • salt=16 字节 强随机
  • hash_len=32 字节
  • 算法：Argon2id

Argon2id 是当前最安全、通用的密码哈希算法，优先替代 bcrypt/PBKDF2。

4、其他算法

SM3：中国国密哈希算法，摘要长度256位，安全性对标SHA-256。
MD5：完全不安全，可碰撞伪造。
SHA-1：已被破解，禁止用于签名。

数字签名

数字签名是消息发送者使用公钥密码算法，产生一段他人无法伪造的数字串，具有不可否认性。消息通过私钥用非对称加密算法对信息进行加密，用私钥加密的信息只能用公钥解密。它结合了非对称加密和消息摘要算法，目的是防篡改、防伪造（只有发送者有私钥）、防抵赖（别人无法伪造你的签名）。

由于非对称算法加解密速度慢，通常先哈希算法对原文计算消息摘要，再用非对称算法的私钥对摘要进行加密获得数字签名，然后把原文和签名发出去，接收方用公钥解密签名，得到摘要后，自己对原文算新的摘要，对比两个摘要是否一致。

数字证书：一段包括用户信息、用户公钥信息以及身份验证机构数字签名的数据。
CA：认证中心，数字证书的申请和签发机构，为公开密钥的用户名称发放证书。
RA：注册审批机构，负责证书申请者的信息录入、审核、发放和管理。

入侵检测技术

入侵检测通过入侵检测系统（IDS）实时监控网络、主机，发现攻击、异常行为并报警。与防火墙不同，IDS主要进行检测、告警和记录，不直接阻断。目前的设备基本都是检测与阻断一体，结合IDS+入侵防御IPS。

IDS主要分为两类：

1、基于网络的入侵检测NIDS

部署再交换机镜像口、网关旁，抓全部流量，查看数据包、协议、攻击特征。

典型场景：端口扫描、暴力破解、SQL注入、XSS、漏洞利用、异常连接。

2、基于主机的入侵检测HIDS

装在服务器/PC本机，监控进程、文件、注册表、日志、系统调用。

典型场景：文件被篡改、后门程序运行、异常提权、敏感文件访问、主机被植入木马。

三种检测原理：

1、特征检测

内置攻击特征库，匹配上就告警。

优点：准确率高、误报少
缺点：认不出新攻击（零日攻击）

2、异常检测

先学习正常行为模型，偏离就告警。

优点：能发现新型攻击
缺点：误报相对多

3、协议分析检测

深度解析协议（HTTP、DNS），发现协议机型、违规操作，常用于对抗缓冲区溢出、构造包攻击。

主流产品

商业：深信服、启明星辰等

开源：Snort（经典IDS），Suricata（高性能多线程）、OSSEC（主机HIDS）、Wazuh（云原生HIDS）。

网络安全协议

网络层

IPSec

对整个IP包加密，有传输和隧道两种模式。如果内部包是IP包，在传输模式下，外部头决定保护内部IP包的IPSec策略；在隧道模式下，内部IP包决定保护其内容的IPSec策略。

两个重要组件：

AH：认证头，保证完整性、身份
ESP：封装安全载荷，提供加密

传输层

1、TLS/SSL

当前主流：TLS1.2/1.3，SSL已不再安全，被TLS逐渐取代。

作用：加密传输内容、验证服务器身份、防止窃听、篡改、中间人攻击。

典型应用：HTTPS = HTTP + TLS

2、SSH（Secure Shell）

远程登陆、文件传输加密。

应用：服务器远程管理、SCP/SFTP文件传输

应用层

1、HTTPS

HTTP+TLS，加密网页浏览，保护账号、密码、支付信息，目前应用广泛。

2、SFTP/SCP

基于SSH的安全文件传输，替代明文的FTP。

3、FTPS

FTP+TLS加密，也是安全文件传输，和SFTP不同的实现。

4、POP3S/IMAPS/SMTPS

邮件协议的加密版本，保护邮箱账号、邮件内容不被窃听。

5、DNSSEC

对DNS记录签名验证，防止DNS污染、域名劫持。

无线安全协议

WPA2/WPA3

WIFI加密标准，WPA2-AES目前最普及，WPA3更安全，抗字典、抗嗅探。

WEP

老式无线加密，早已被破解，不安全。
