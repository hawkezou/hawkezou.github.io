访问Github失败处理

五月 5, 2026 技术 本文总阅读量次

国内访问GitHub大多是 DNS 解析污染/超时，优先用修改Hosts → 换公共 DNS → 镜像/代理，基本都能解决。

快速排查

  1. 查 GitHub 状态：https://www.githubstatus.com（看官方是否宕机）。
  2. 换浏览器/无痕模式，关闭所有插件（尤其广告拦截、隐私类）。
  3. 重启路由器/电脑，切换手机 4G/5G 对比（判断是宽带问题还是全局问题）。
  4. 临时关闭防火墙/杀毒软件（测试是否被拦截）。

方法1：修改 Hosts

原理：直接绑定GitHub 最新IP，绕过DNS污染。

（1）获取最新 IP（任选其一）：

  • 访问：https://github.com.ipaddress.com/github.com
  • 访问：https://ping.chinaz.com/github.com
  • cmd 执行：ping github.com，抄下IP。

（2）编辑 Hosts 文件：

  • Windows：管理员打开记事本 → 打开 C:\Windows\System32\drivers\etc\hosts。
  • MacOS/Linux：终端输入 sudo nano /etc/hosts，输密码编辑。

  • Windows cmd：ipconfig /flushdns。
  • MacOS：sudo killall -HUP mDNSResponder。
  • Linux：sudo systemd-resolve --flush-caches。

重新访问 github.com。

懒人工具：用 SwitchHosts 或 FastGithub，一键自动更新最优 Hosts，无需手动查 IP。

方法2：换公共 DNS

把 DNS 改成国内/国际公共服务器：

  • Windows：网络设置 → 更改适配器选项 → 右键 Wi-Fi/以太网 → 属性 → IPv4 → 手动 DNS：

首选：114.114.114.114（国内） 备选：8.8.8.8（Google）或 1.1.1.1（Cloudflare）。

  • MacOS：系统设置 → 网络 → 高级 → DNS，添加上述 IP。改完后刷新 DNS 缓存（同方法1第4步）。
